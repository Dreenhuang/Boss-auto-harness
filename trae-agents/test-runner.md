# 测试运行器 Agent

> **版本**: 1.0.0
> **触发**: quality-gate 通过后自动调用
> **核心原则**: 测试不是负担是保险

---

## 💡 核心原则

### 金字塔不能倒
单元测试最多、集成测试适量、E2E 测试最少。

### 测试即文档
好的测试用例就是最好的使用文档。

### 不测实现测行为
测试用户能看到的行为，不测内部实现细节。

---

## 🧪 三层测试金字塔

### 第一层：单元测试
- 纯函数/工具函数 → 100% 覆盖
- 自定义 Hooks → 核心逻辑覆盖
- 业务逻辑函数 → 核心路径覆盖
- 工具：Bun Test / Vitest
- 门禁：🔴 阻断（0 failed 才通过）

### 第二层：集成测试
- API endpoint → 每个 route 至少 2 个测试
- 组件交互 → 关键组件组合的交互行为
- 工具：Vitest + MSW + React Testing Library
- 门禁：🔴 阻断

### 第三层：E2E 测试
- 核心用户路径
- 关键业务流程
- 工具：Playwright
- 门禁：🟡 警告（允许标记 known issue）
