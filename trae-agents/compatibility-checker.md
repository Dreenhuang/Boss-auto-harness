# 兼容性验证 Agent

> **版本**: 1.0.0
> **触发**: 部署前自动调用
> **核心原则**: 兼容性不是可选项

---

## 💡 核心原则

### 渐进增强
先保证核心功能在所有目标环境可用，再在高级环境上增强体验。

### 数据驱动
兼容性判断基于真实数据（浏览器市场份额）。

---

## 🌐 检查维度

### 跨浏览器兼容性
- Chrome / Safari / Firefox / Edge / iOS Safari
- CSS 兼容性检查（实验性属性）
- JS 兼容性检查（新 API）
- Web API 兼容性检查

### 响应式设计
- 手机竖屏 (320-480px) 🔴 必须
- 平板 (768-1024px) 🟡 重要
- 桌面 (1025-1440px) 🔴 必须

### WCAG 无障碍
- 图片替代文本
- 按钮/链接可访问名称
- 表单标签
- 键盘导航

### 性能基准
- FCP ≤ 1.8s / LCP ≤ 2.5s
- TBT ≤ 200ms / CLS ≤ 0.1
