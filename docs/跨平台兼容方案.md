## 三、兼容方案设计

### 3.1 方案一：格式转换导出器（推荐）

创建一个自动导出脚本，将 Boss-auto-harness 的完整配置转换为目标平台格式：

#### 支持的输出格式

| 目标平台 | 输出路径 | 转换规则 |
|---------|----------|---------|
| Cursor | `.cursor/rules/boss-mode.md` | 直接复制 CLAUDE.md |
| Windsurf | `.windsurf/rules/boss-mode.md` | 直接复制 CLAUDE.md |
| VS Code Copilot | `.github/copilot-instructions.md` | 提取核心规则部分 |
| Devin | `devin.yaml` | YAML 格式转换 |
| Claude Code | `CLAUDE.md` | 完整复制 |
| Gemini CLI | `.gemini/instructions.md` | 提取核心规则部分 |

#### 导出脚本实现

```powershell
# export-to-platform.ps1
param(
    [Parameter(Mandatory=$true)]
    [ValidateSet("cursor", "windsurf", "vscode", "devin", "claude", "gemini")]
    [string]$Platform,
    
    [string]$TargetDir = "."
)

$SourceCLAUDE = "CLAUDE.md"
$content = Get-Content $SourceCLAUDE -Raw -Encoding UTF8

switch ($Platform) {
    "cursor" {
        $destDir = Join-Path $TargetDir ".cursor/rules"
        New-Item -ItemType Directory -Path $destDir -Force | Out-Null
        Set-Content -Path (Join-Path $destDir "boss-mode.md") -Value $content -Encoding UTF8
        Write-Host "✅ Exported to Cursor: $destDir/boss-mode.md"
    }
    "windsurf" {
        $destDir = Join-Path $TargetDir ".windsurf/rules"
        New-Item -ItemType Directory -Path $destDir -Force | Out-Null
        Set-Content -Path (Join-Path $destDir "boss-mode.md") -Value $content -Encoding UTF8
        Write-Host "✅ Exported to Windsurf: $destDir/boss-mode.md"
    }
    "vscode" {
        $destDir = Join-Path $TargetDir ".github"
        New-Item -ItemType Directory -Path $destDir -Force | Out-Null
        # 提取核心规则部分
        $copilotContent = Extract-CopilotRules $content
        Set-Content -Path (Join-Path $destDir "copilot-instructions.md") -Value $copilotContent -Encoding UTF8
        Write-Host "✅ Exported to VS Code Copilot: $destDir/copilot-instructions.md"
    }
    "devin" {
        # 转换为 YAML 格式
        $devinContent = ConvertTo-DevinYaml $content
        Set-Content -Path (Join-Path $TargetDir "devin.yaml") -Value $devinContent -Encoding UTF8
        Write-Host "✅ Exported to Devin: $TargetDir/devin.yaml"
    }
    "claude" {
        $destDir = Join-Path $TargetDir ".claude"
        New-Item -ItemType Directory -Path $destDir -Force | Out-Null
        Copy-Item -Path $SourceCLAUDE -Destination (Join-Path $destDir "CLAUDE.md") -Force
        Write-Host "✅ Exported to Claude Code: $destDir/CLAUDE.md"
    }
    "gemini" {
        $destDir = Join-Path $TargetDir ".gemini"
        New-Item -ItemType Directory -Path $destDir -Force | Out-Null
        $geminiContent = Extract-GeminiRules $content
        Set-Content -Path (Join-Path $destDir "instructions.md") -Value $geminiContent -Encoding UTF8
        Write-Host "✅ Exported to Gemini CLI: $destDir/instructions.md"
    }
}
```

```markdown

8. Git Commit → Automated version control
9. Deploy → Multi-target deployment

## Trigger Keywords
- "Boss", "boss模式", "boss moshi", "Boss-harness", "harness", "全自动开发"

- 0 critical/high security issues

## Best Practices
- MVP first
- Small iterative steps
- Full traceability
- No hard-coded secrets
```

## 四、Skills 跨平台适配

### 4.1 Skill 格式标准化

当前 Skill 使用 Markdown 格式定义，具有天然的跨平台兼容性：

```
skills/
├── product-spec-builder/
│   └── SKILL.md          # 通用格式，可直接复制
├── dev-builder/
│   └── SKILL.md
└── ...
```

### 4.2 平台特定适配层

对于不支持 Markdown 规则的平台，创建适配层：

```
adapters/
├── cursor/
│   └── boss-mode.json     # Cursor 规则 JSON 格式
├── vscode/
│   └── copilot-settings.json
└── devin/
    └── devin-config.yaml
```

## 七、限制与注意事项

### 7.1 功能差异

| 功能 | TRAE (完整) | Cursor | Windsurf | VS Code | Devin |
|------|------------|--------|----------|---------|-------|
| 自动 Skill 调用 | ✅ | ⚠️ | ⚠️ | ❌ | ❌ |
| Sub-Agent 派发 | ✅ | ❌ | ❌ | ❌ | ✅ |
| Pipeline 编排 | ✅ | ❌ | ❌ | ❌ | ⚠️ |
| 自动 Git 提交 | ✅ | ❌ | ❌ | ❌ | ✅ |
| 质量门禁 | ✅ | ⚠️ | ⚠️ | ❌ | ⚠️ |

### 7.2 建议

1. **最佳体验**: 使用 TRAE IDE + Claude 模型
2. **次优体验**: 使用 Cursor + Claude/GPT-4
3. **基础体验**: 其他平台可使用核心规则，但自动化能力有限
