## 一、管理后台概述

### 1.1 定位与目标

Vibe PM 管理后台是面向平台运营人员、内容管理员和系统管理员的一站式运营管理平台，旨在提供高效的用户管理、内容运营、数据分析和系统配置能力。

### 1.2 目标用户

| 角色 | 职责 | 权限级别 |
|------|------|----------|
| 超级管理员 | 系统配置、权限分配、全局管理 | 最高 |
| 运营经理 | 用户管理、内容审核、活动策划 | 高 |
| 内容编辑 | 内容发布、学习路径管理、卡片维护 | 中 |
| 数据分析师 | 数据查看、报表导出、趋势分析 | 中 |
| 客服专员 | 用户反馈处理、投诉管理、消息回复 | 低 |

## 二、功能模块规划

### 模块一：仪表盘（Dashboard）

**优先级：P0（最高）**

#### 核心功能点

| 功能 | 描述 | 业务价值 |
|------|------|----------|
| 核心数据概览 | 实时展示用户总数、日活、新增、留存率 | 快速掌握平台运营状况 |
| 数据趋势图 | 7/30/90天用户增长、活跃度趋势 | 发现增长规律和异常 |
| 热门内容排行 | 展示最受欢迎的卡片、路径、话题 | 指导内容运营方向 |
| 待办事项提醒 | 待审核内容、待处理反馈、系统告警 | 提高运营效率 |
| 快捷操作入口 | 一键发布内容、批量审核、导出数据 | 简化日常操作 |

#### 数据指标

```
- 总用户数 / 今日新增 / 今日活跃
- 总内容数 / 今日发布 / 待审核
- 平均学习时长 / 完成率 / 收藏转化率
- 系统健康度 / API响应时间 / 错误率
```

### 模块二：用户管理

**优先级：P0（最高）**

#### 2.1 用户列表

| 功能 | 描述 |
|------|------|
| 用户搜索 | 支持按手机号、昵称、ID、注册时间搜索 |
| 高级筛选 | 按角色、状态、注册渠道、活跃度筛选 |
| 批量操作 | 批量启用/禁用、批量发送通知、批量导出 |
| 用户详情 | 查看用户完整信息、学习记录、操作日志 |
| 用户标签 | 给用户打标签（高活跃用户、潜在流失等） |

#### 2.2 用户详情页

```
├─ 基本信息（头像、昵称、手机号、注册时间）
├─ 学习数据（学习天数、完成路径、收藏数）
├─ 行为记录（登录日志、操作记录、浏览历史）
├─ 内容贡献（发布的笔记、评论、点赞记录）
├─ 账号状态（启用/禁用、角色、权限）
└─ 操作按钮（重置密码、发送通知、编辑信息）
```

#### 2.3 角色与权限

| 功能 | 描述 |
|------|------|
| 角色管理 | 创建/编辑/删除角色，定义角色权限 |
| 权限配置 | 细粒度权限控制（页面访问、操作权限、数据范围） |
| 管理员列表 | 管理后台管理员账号的增删改查 |
| 操作日志 | 记录所有管理员的操作行为，支持审计 |

### 模块三：内容管理

**优先级：P0（最高）**

#### 3.1 学习卡片管理

| 功能 | 描述 |
|------|------|
| 卡片列表 | 展示所有学习卡片，支持搜索和筛选 |
| 卡片编辑 | 富文本编辑器，支持Markdown、图片、代码块 |

| 卡片分类 | 管理分类标签，调整分类层级 |
| 卡片推荐 | 设置首页推荐卡片，调整展示顺序 |

#### 3.2 学习路径管理

| 功能 | 描述 |
|------|------|
| 路径列表 | 管理所有学习路径 |
| 路径编辑 | 设置路径名称、描述、封面、步骤 |
| 步骤管理 | 添加/删除/排序路径中的学习步骤 |
| 路径审核 | 审核用户创建的学习路径 |
| 路径推荐 | 设置推荐路径，调整展示权重 |

#### 3.3 内容审核

| 功能 | 描述 |
|------|------|
| 审核队列 | 待审核的内容列表（卡片、路径、评论） |
| 批量审核 | 批量通过/驳回内容 |
| 审核规则 | 设置自动审核规则（敏感词、图片检测） |
| 审核记录 | 查看历史审核记录和审核统计 |

### 模块四：数据统计分析

**优先级：P1（高）**

#### 4.1 用户分析

| 功能 | 描述 |
|------|------|
| 用户增长 | 新增用户趋势、渠道分析、留存分析 |
| 用户画像 | 年龄分布、地域分布、设备分布、兴趣标签 |
| 活跃度分析 | DAU/WAU/MAU、使用时长、访问频次 |

#### 4.2 内容分析

| 功能 | 描述 |
|------|------|

| 分类分析 | 各分类内容量、热度、用户偏好 |
| 搜索分析 | 热搜词、搜索转化率、无结果搜索 |

#### 4.3 学习分析

| 功能 | 描述 |
|------|------|
| 学习进度 | 整体学习完成率、平均学习时长 |
| 路径效果 | 各路径的完成率、放弃率、满意度 |
| 知识掌握 | 用户知识点掌握程度分布 |
| 学习行为 | 学习时段分布、学习偏好、断点续学率 |

#### 4.4 报表导出

| 功能 | 描述 |
|------|------|
| 自定义报表 | 选择指标、时间范围、维度组合生成报表 |
| 定时报表 | 设置日报/周报/月报自动发送 |
| 数据导出 | 支持Excel、CSV、PDF格式导出 |

## 三、技术架构建议

### 3.1 前端架构

```
vibePM-admin/
├── src/
│   ├── views/           # 页面组件
│   │   ├── Dashboard/
│   │   ├── UserManage/
│   │   ├── ContentManage/
│   │   ├── DataAnalysis/
│   │   ├── SystemConfig/
│   │   └── Operation/
│   ├── components/      # 公共组件
│   │   ├── DataTable/   # 数据表格
│   │   ├── Charts/      # 图表组件
│   │   ├── SearchFilter/# 搜索筛选
│   │   └── Modal/       # 弹窗组件
│   ├── stores/          # Pinia状态管理
│   ├── api/             # API接口
│   └── utils/           # 工具函数
├── public/
└── package.json
```

### 3.2 技术选型

| 技术 | 选型 | 说明 |
|------|------|------|
| 框架 | Vue 3 + Vite | 与前端项目保持一致 |
| UI库 | Element Plus | 适合管理后台的组件库 |
| 图表 | ECharts | 丰富的数据可视化能力 |
| 状态管理 | Pinia | 与前端项目保持一致 |
| 路由 | Vue Router | 支持权限路由控制 |
| 构建 | Vite | 快速构建和热更新 |

### 3.3 权限控制

```javascript
// 路由权限控制
const routes = [
  {
    path: '/user',
    component: UserManage,
    meta: {
      requiresAuth: true,
      permissions: ['user:view']
    }
  }
]

// 按钮级权限
<v-if="hasPermission('user:edit')">
  <el-button>编辑用户</el-button>
</v-if>
```

## 四、开发优先级排序

### Phase 1（第1-2周）：基础框架 + 核心功能

| 优先级 | 模块 | 功能 |
|--------|------|------|
| P0 | 框架搭建 | 项目初始化、路由配置、权限系统、布局框架 |
| P0 | 仪表盘 | 核心数据概览、数据趋势图 |
| P0 | 用户管理 | 用户列表、用户详情、搜索筛选 |

### Phase 2（第3-4周）：内容管理 + 数据展示

| 优先级 | 模块 | 功能 |
|--------|------|------|
| P0 | 内容管理 | 卡片管理、路径管理、内容审核 |
| P1 | 数据统计 | 用户分析、内容分析、基础报表 |

### Phase 3（第5-6周）：系统配置 + 运营工具

| 优先级 | 模块 | 功能 |
|--------|------|------|
| P1 | 系统配置 | 基础配置、内容配置、通知配置 |
| P2 | 运营管理 | 活动管理、消息推送、反馈管理 |

### Phase 4（第7-8周）：高级功能 + 优化

| 优先级 | 模块 | 功能 |
|--------|------|------|
| P1 | 数据深化 | 学习分析、留存分析、自定义报表 |
| P1 | 系统监控 | 服务状态、性能监控、日志查看 |
| P2 | 优化迭代 | 性能优化、交互优化、Bug修复 |

## 五、接口规划

### 5.1 用户管理接口

```
GET    /api/admin/users              # 用户列表
GET    /api/admin/users/:id          # 用户详情
PUT    /api/admin/users/:id          # 更新用户
DELETE /api/admin/users/:id          # 删除用户
POST   /api/admin/users/:id/ban      # 禁用用户
POST   /api/admin/users/:id/unban    # 启用用户
POST   /api/admin/users/batch        # 批量操作
```

### 5.2 内容管理接口

```
GET    /api/admin/posts              # 卡片列表
GET    /api/admin/posts/:id          # 卡片详情
PUT    /api/admin/posts/:id          # 更新卡片
DELETE /api/admin/posts/:id          # 删除卡片
POST   /api/admin/posts/:id/audit    # 审核卡片
GET    /api/admin/paths              # 路径列表
PUT    /api/admin/paths/:id          # 更新路径
```

### 5.3 数据统计接口

```
GET    /api/admin/stats/overview     # 概览数据
GET    /api/admin/stats/users        # 用户统计
GET    /api/admin/stats/content      # 内容统计
GET    /api/admin/stats/learning     # 学习统计
GET    /api/admin/stats/export       # 导出报表
```

### 5.4 系统配置接口

```
GET    /api/admin/config             # 获取配置
PUT    /api/admin/config             # 更新配置
GET    /api/admin/roles              # 角色列表
POST   /api/admin/roles              # 创建角色
PUT    /api/admin/roles/:id          # 更新角色
```

## 八、风险评估

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 数据量大导致性能问题 | 高 | 分页加载、虚拟滚动、数据缓存 |
| 权限控制漏洞 | 高 | 前后端双重校验、操作审计 |
| 第三方图表库兼容性 | 中 | 预留替换方案、充分测试 |
| 开发进度延迟 | 中 | 分阶段交付、优先核心功能 |
