### 1.2 目标用户群体

| 用户类型 | 特征 | 核心需求 |
|----------|------|----------|
| **技术小白** | 零基础，想转行产品经理或学习技术 | 通俗易懂的内容、清晰的学习路径 |

### 1.3 整体业务目的

> **一句话**: AI时代产品经理的学习神器——像刷小红书一样掌握技术思维

- **内容层**: 提供高质量、结构化的技术学习内容

- **社区层**: 点赞、收藏、学习路径，形成学习闭环
- **商业层**: 未来可扩展付费课程、企业培训

## 二、AI大模型定时批量生成内容特性分析

### 2.1 当前实现

后端已集成 MiniMax API：
- **图片生成**: 每日50张配额，根据技术主题生成概念图
- **内容更新**: 定时自动更新机制（待完善）

### 2.2 AI生成内容特性

| 特性 | 说明 | 对发布功能的影响 |
|------|------|-----------------|
| **批量性** | AI可同时生成多篇内容 | 需要批量审核、批量发布机制 |
| **自动性** | 定时触发，无需人工干预 | 需要自动发布工作流 + 人工复核开关 |
| **主题性** | 内容围绕特定技术栈/工具 | 需要分类标签自动匹配 |
| **质量波动** | AI生成质量不稳定 | 需要审核流程，支持草稿/待审核/已发布状态 |
| **图片关联** | 每张卡片需要配图 | 需要图片生成与内容绑定的机制 |

### 3.2 功能模块说明

#### 模块1: 内容创建

| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| **手动创建** | 管理员在后台填写标题、内容、分类、标签 | P0 |
| **AI辅助生成** | 调用MiniMax/DeepSeek生成内容摘要、标题优化 | P1 |
| **批量导入** | 支持Markdown/JSON批量导入内容 | P2 |
| **图片上传** | 本地上传或AI生成配图 | P0 |

#### 模块2: 审核工作流

| 状态 | 说明 | 流转条件 |
|------|------|----------|
| **draft** | 草稿，仅管理员可见 | 创建后默认状态 |
| **pending** | 待审核，进入审核队列 | 提交审核后 |
| **approved** | 已通过审核，待发布 | 审核通过后 |
| **published** | 已发布，前端可见 | 到达发布时间后 |
| **rejected** | 已驳回，需修改 | 审核不通过 |
| **archived** | 已归档，前端不可见 | 手动下架 |

#### 模块3: 定时发布

| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| **定时设置** | 选择未来某个时间点自动发布 | P0 |
| **批量定时** | 一次性设置多篇内容的发布时间 | P1 |
| **发布队列** | 查看即将发布的内容列表 | P1 |
| **取消发布** | 在发布前取消定时任务 | P1 |

#### 模块4: AI批量生成集成

| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| **提示词模板** | 从提示词管理中选择生成模板 | P0 |
| **批量生成** | 一次性生成多篇内容草稿 | P1 |
| **自动配图** | 根据内容主题自动调用图片生成 | P1 |
| **质量评分** | AI自动评估内容质量，低质量标记待审 | P2 |

```sql
ALTER TABLE posts ADD COLUMN IF NOT EXISTS scheduled_at TIMESTAMPTZ;

ALTER TABLE posts ADD COLUMN IF NOT EXISTS reviewed_by INTEGER REFERENCES users(id);

ALTER TABLE posts ADD COLUMN IF NOT EXISTS reject_reason TEXT;
ALTER TABLE posts ADD COLUMN IF NOT EXISTS ai_generated BOOLEAN DEFAULT FALSE;
ALTER TABLE posts ADD COLUMN IF NOT EXISTS prompt_key VARCHAR(100); -- 使用的提示词模板

-- 发布任务表（用于定时发布）
CREATE TABLE IF NOT EXISTS publish_tasks (
  id SERIAL PRIMARY KEY,
  post_id INTEGER REFERENCES posts(id),
  scheduled_at TIMESTAMPTZ NOT NULL,
  status VARCHAR(20) DEFAULT 'pending', -- pending/completed/failed/cancelled
  executed_at TIMESTAMPTZ,
  error_message TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 内容审核日志
CREATE TABLE IF NOT EXISTS content_audit_logs (
  id SERIAL PRIMARY KEY,
  post_id INTEGER REFERENCES posts(id),
  action VARCHAR(50) NOT NULL, -- submit/approve/reject/publish
  operator_id INTEGER,
  operator_name VARCHAR(100),
  reason TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 3.4 API接口设计

```
# 内容管理（扩展现有）
POST   /api/admin/posts              # 创建内容
PUT    /api/admin/posts/:id          # 更新内容
POST   /api/admin/posts/:id/submit   # 提交审核
POST   /api/admin/posts/:id/audit    # 审核内容 {status, reason}
POST   /api/admin/posts/:id/schedule # 定时发布 {scheduled_at}
POST   /api/admin/posts/batch-audit  # 批量审核

# AI批量生成
POST   /api/admin/posts/ai-generate  # AI生成内容草稿 {prompt_key, count, category}
POST   /api/admin/posts/batch-create # 批量创建

# 发布任务
GET    /api/admin/publish-tasks      # 获取发布任务列表
DELETE /api/admin/publish-tasks/:id  # 取消发布任务
```

## 四、实施建议

### 4.1 开发优先级

| 阶段 | 功能 | 预计工时 | 依赖 |
|------|------|----------|------|
| **Phase 1** | 内容创建/编辑弹窗 + 状态管理 | 4h | 无 |
| **Phase 2** | 审核工作流（提交/通过/驳回） | 3h | Phase 1 |
| **Phase 3** | 定时发布 + 任务队列 | 3h | Phase 2 |
| **Phase 4** | AI批量生成集成 | 4h | Phase 3 |
| **Phase 5** | 前端用户端API真实数据对接 | 3h | 无 |
| **Phase 6** | 添加更多提示词模板 | 2h | 无 |
| **Phase 7** | AI图片生成调用提示词模板 | 3h | Phase 6 |

### 4.2 技术实现要点

1. **定时任务**: 使用 `node-cron` 每分钟检查待发布的任务
2. **状态机**: 使用状态模式管理内容生命周期
3. **并发控制**: 批量生成时控制并发数，避免API限流
4. **容错处理**: AI生成失败时记录日志，支持重试
