## 2. 前置知识：什么是救猫咪理论

### 2.1 理论起源

布莱克·斯奈德（Blake Snyder）是好莱坞著名编剧，他将自己20年的编剧经验总结成《Save the Cat!》系列书籍，成为好莱坞最受欢迎的编剧指南之一。

**核心观点：**
> "每个电影都在承诺一种体验。观众买票进场，是因为你承诺了某种满足。"

### 2.2 理论核心

救猫咪理论包含三大支柱：

```
救猫咪理论三大支柱
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐   │
│    │  15节拍表   │ +  │  10种类型   │ +  │  角色弧光   │   │
│    │  故事结构   │    │  类型规则   │    │  人物设计   │   │
│    └─────────────┘    └─────────────┘    └─────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 为什么选择这个工具？

| 传统方式 | 使用救猫咪编剧助手 |
|---------|------------------|
| 需要阅读大量书籍 | 直接实操，在做中学 |
| 理论难以落地 | AI辅助，生成完整剧本 |
| 容易遗漏细节 | 系统化流程，确保完整 |
| 创作瓶颈难突破 | AI提供思路和选项 |

### 3.2 两种创作模式对比

| 维度 | 直接生成模式 | 共创协作模式 |
|------|------------|-------------|
| **适合场景** | 有明确想法 | 想法模糊/新手 |
| **交互深度** | 浅，1-2轮 | 深，5-10轮 |
| **时间成本** | 快速，5-10分钟 | 较长，30-60分钟 |
| **创作质量** | 依赖用户想法质量 | 更深入、更个性化 |
| **学习价值** | 中等 | 高，过程中学习理论 |

### 3.3 触发词说明

在AI对话中输入以下任一触发词，即可激活救猫咪编剧助手：

```
救猫咪      ← 最常用
写剧本
编剧助手
```

**注意：** 不同AI助手可能有不同的激活方式，请参考具体平台的说明。

## 5. 模式一：直接生成模式详解

### 5.1 模式说明

直接生成模式适合**有明确故事想法**的用户。你只需要提供一句话概念，AI会自动完成分析、类型匹配、结构生成等工作。

### 5.2 完整操作流程

#### 5.2.1 Step 1: 输入故事概念

**好的概念示例：**

| 概念类型 | 示例 | 质量 |
|---------|------|------|
| 完整概念 | "一个离婚律师在一次意外后发现自己有了超能力，必须重新学会如何与人亲近" | ⭐⭐⭐⭐⭐ |
| 中等概念 | "一个律师有了超能力，会发生什么" | ⭐⭐⭐ |
| 模糊概念 | "我想写一个超能力的故事" | ⭐⭐ |

**概念公式：**
```
一个[主角身份]想要[目标]，但是[障碍]
```

**示例：**
```
一个[失忆的特工]想要[找回身份]，但是[被通缉]
一个[过气的魔术师]想要[重振事业]，但是[必须和不懂魔术的经纪人合作]
一个[普通的上班族]想要[逃离无聊的生活]，但是[被困在时间循环里]
```

### 6.3 对话示例

#### 第一轮：故事种子

```
AI：你好！让我们一起创作你的故事。

第一轮问题：
用一个画面或一句话告诉我，你脑海中最初的故事画面是什么？

格式引导："一个（主角）想要（目标），但是（障碍）"
或者直接描述你想象中的场景。
```

**你的回答选项：**

| 选项 | 回答方式 |
|------|---------|
| A | 直接描述想法："一个普通人突然获得了超能力" |
| B | 说没想法："我还没想好" |
| C | 描述模糊画面："我想象一个人在黑暗中奔跑" |

#### AI自动生成选项（当你没想法时）

```
当你回答"我没想法"时，AI会自动生成3-5个高质量选项：

AI：没问题！让我给你几个方向：

选项A：一个失忆的CIA特工醒来，发现自己被通缉
→ 适合：高节奏、动作导向、身份危机
→ 案例参考：《谍影重重》《记忆碎片》
→ 优势：开篇即有强戏剧性

选项B：退休特工突然失忆，被迫重新面对自己的过去
→ 适合：情感深度、角色驱动
→ 案例参考：《内陆帝国》《都市日落》
→ 优势：更容易建立情感共鸣

...

请选择一个方向：A / B / C / D
```

### 6.4 智能选项生成机制

当你说"不知道""没想法"时，AI的选项生成遵循以下原则：

| 生成原则 | 说明 | 示例 |
|---------|------|------|
| **相关性** | 选项与当前讨论主题相关 | 都涉及特工/失忆 |
| **多样性** | 选项之间有明显的方向差异 | 动作型vs情感型 |
| **具体性** | 每个选项都有清晰的描述 | 不是泛泛的概念 |
| **价值说明** | 包含适合场景和案例 | 告诉用户为什么选 |
| **可选性** | 用户可以真实地选择任何一个 | 不是唯一答案 |

```
│  ├── 节拍10: 坏人逼近                                       │

│  └── 节拍12: 灵魂黑夜                                       │
│                                                             │
│  【第三幕：解决】第75-110页                                  │
│  ├── 节拍13: 第三幕衔接点                                    │
│  ├── 节拍14: 结局                                           │
│  └── 节拍15: 终场画面                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 7.2 第一幕详解

#### 节拍1：开场画面（第1页）

**定义：** 观众看到的第一个画面

**功能：**
- 建立电影的基调和风格
- 展示主角是谁、处境如何
- 给观众一个"承诺"

**案例：**
- 《星球大战》：莱娅公主被达斯·维达押送
- 《阿甘正传》：羽毛飘落，阿甘坐在长椅上

**设计要点：**
```
开场画面 = 一个画面 + 主角是谁 + 情绪基调
```

#### 节拍2：主题阐明（第5页）

**定义：** 某个角色说出故事的主题

**功能：**
- 告诉观众电影"关于什么"
- 主角此时不理解，但观众开始理解
- 为B故事埋下伏笔

**案例：**
- 《狮子王》：木法沙说"阳光下都有阴影"
- 《阿甘正传》：母亲说"生命就像一盒巧克力"

**设计要点：**
```
主题 ≠ 说教，而是对生活的观察
主角此时会忽略或嘲笑这个观点
```

#### 节拍3：铺垫（第1-10页）

**定义：** 展示主角的"旧世界"

**核心任务：**
- 让观众了解主角是谁
- 建立主角的缺陷
- 建立"救猫咪时刻"

**救猫咪时刻：** 主角做一个"可爱"的行为，让观众喜欢主角

**设计要点：**
```
铺垫 = 主角的日常生活 + 缺陷展示 + 救猫咪时刻
```

#### 节拍4：催化剂（第12页）

**定义：** 生活被打断的事件

**功能：**
- 彻底打破主角的平衡
- 给主角一个"钩子"
- 从这里开始，主角无法回头

**设计要点：**
```
催化剂 = 一个事件 + 完全改变主角处境 + 意外但合理
```

#### 节拍5：争执（第12-25页）

**定义：** 主角质疑"我要不要去做？"

**核心任务：**
- 展示内心冲突
- 让观众理解动机
- 为进入"新世界"做心理准备

**设计要点：**
```
争执 = "这能行吗？"的疑问 + 考虑利弊 + 导师建议
```

#### 节拍6：第二幕衔接点（第25页）

**定义：** 主角做出决定，进入"新世界"

**标志性时刻：** "好吧，我去做！"

**设计要点：**
```
衔接点 = 主动选择 + 进入新世界 + 标志性台词
```

### 7.3 第二幕详解

#### 节拍7：B故事（第30页）

**定义：** 爱情/友谊故事线

**功能：**
- 提供情感休憩
- 展示真正的主题
- 建立关系线

**与A故事的关系：**
- A故事：发生了什么
- B故事：这意味着什么

**设计要点：**
```
B故事 = 爱情/友谊/师徒 + 主题探讨 + 给主角启示
```

#### 节拍8：游戏时间（第30-55页）

**定义：** "承诺兑现"的时刻

**功能：**
- 观众来电影院看的就是这个
- 展示主角尝试不同方法
- 建立故事基调

**设计要点：**
```
游戏时间 = 最有趣的部分 + 追逐/战斗/浪漫 + 升级的障碍
```

#### 节拍9：中点（第55页）

**定义：** 假胜利或假失败

**两种类型：**
| 类型 | 说明 | 案例 |
|------|------|------|
| 假胜利 | 主角似乎达到了目标 | 死星被摧毁 |
| 假失败 | 主角似乎失去了一切 | 沉船后杰克沉入大海 |

**设计要点：**
```
中点 = 提高赌注 + 让观众以为"差不多了" + 改变走向
```

#### 节拍10：坏人逼近（第55-75页）

**定义：** 压力越来越大

**压力来源：**
- 外部：敌人越来越强
- 内部：自我怀疑
- 关系：与盟友出现裂痕

**设计要点：**
```
坏人逼近 = 外压 + 内压 + 时间压力 + 盟友动摇
```

#### 节拍11：一无所有（第75页）

**定义：** "死亡气息"

**核心：** 主角失去了最重要的东西

**"Whiff of Death"：** 在"死亡"之前的最后温馨

**设计要点：**
```
一无所有 = 失去盟友/爱人 + "死亡气息" + 预警信号
```

#### 节拍12：灵魂黑夜（第75-85页）

**定义：** 故事的最低点

**与"一无所有"的区别：**
- 一无所有：外在的（失去什么）
- 灵魂黑夜：内在的（信心崩溃）

**设计要点：**
```
灵魂黑夜 = 彻底失败 + 质疑一切 + "为什么？" + 故事指向失败
```

### 7.4 第三幕详解

#### 节拍13：第三幕衔接点（第85页）

**定义：** 主角获得洞察/启示

**灵感来源：**
- B故事角色的某句话
- 一个意外的帮助
- 对过去的重新审视

**设计要点：**
```
衔接点 = 洞察 + 知道真正需要改变什么 + 准备最后一战
```

#### 节拍14：结局（第85-110页）

**定义：** 最后一战

**子节拍：**
1. 冲向最终对决
2. 最接近失败
3. 使用学到的智慧获胜
4. 外在胜利
5. 新秩序建立

**设计要点：**
```
结局 = 最伟大的场景 + 展示成长 + 问题解决 + 新秩序
```

#### 节拍15：终场画面（第110页）

**定义：** 与开场画面形成对比

**功能：**
- 证明故事确实发生了变化
- 给观众最终的情感满足
- 最后的情绪印记

**设计要点：**
```
终场 = 与开场对比 + 证明改变 + 情绪印记
```

```
├─ 爱情/友情（两个不完整的人） → 伙伴之情型
│
├─ 谜题破解（真相/秘密） → 推理侦探型
│
├─ 讽刺智慧（傻瓜战胜聪明人） → 傻瓜获胜型
```

## 9. 角色弧光设计教程

### 9.1 什么是角色弧光？

角色弧光是角色在故事中的**内在转变**。它不是角色"做了什么"，而是角色"变成了什么"。

```
角色弧光 = 开始状态 → 转变 → 结束状态
```

### 9.2 角色弧光三要素

| 要素 | 定义 | 问题 |
|------|------|------|
| **缺陷** | 角色开始时的性格问题 | "角色最讨厌自己什么？" |
| **欲望** | 角色追求的外在目标 | "角色想要什么？" |
| **需求** | 角色真正需要的内在成长 | "角色真正需要什么？" |

### 9.3 缺陷设计

**缺陷类型：**

| 类型 | 示例 | 电影案例 |
|------|------|---------|
| 道德缺陷 | 自私、贪婪、残忍 | 《星游记》达斯·维达 |
| 心理缺陷 | 恐惧、创伤、偏执 | 《搏击俱乐部》杰克 |
| 社会缺陷 | 傲慢、冷漠、偏见 | 《窈窕淑女》亨利 |
| 情感缺陷 | 害怕亲密、无法信任 | 《当哈里遇见莎莉》哈里 |

**设计缺陷的原则：**
1. 缺陷必须在故事开始时清晰可见
2. 缺陷必须是角色自己可以改变的
3. 缺陷必须影响角色的决定

### 9.4 欲望 vs 需求

| 维度 | 欲望（Want） | 需求（Need） |
|------|------------|------------|
| 意识 | 有意识追求 | 无意识需要 |
| 表达 | "我想要..." | "我需要..." |
| 实现 | 实现后不一定满足 | 必须满足才能完整 |
| 与主题 | 外在追求 | 内在成长 |

**案例对比：**

| 电影 | 欲望 | 需求 |
|------|------|------|
| 《阿甘正传》 | 珍妮的爱 | 学会爱自己 |
| 《王牌天神》 | 成为上帝 | 学会关心他人 |
| 《当哈里遇见莎莉》 | 证明男女不能做朋友 | 学会信任 |
| 《蜘蛛侠》 | 被认可 | 承担责任 |

### 9.5 弧光与节拍对应

```
开场画面 → 缺陷展示
    ↓
催化剂 → 欲望出现
    ↓
争执 → 抗拒改变
    ↓
游戏时间 → 用旧方式追求欲望（不断失败）
    ↓
中点 → 发现新可能
    ↓
一无所有 → 旧方式彻底失败
    ↓
灵魂黑夜 → 质疑一切
    ↓
第三幕衔接点 → 意识到需求
    ↓
结局 → 用新方式满足需求
    ↓
终场画面 → 展示新自我
```

## 10. 导出与格式转换

### 10.1 支持的格式

| 格式 | 扩展名 | 优点 | 缺点 |
|------|--------|------|------|
| Markdown | .md | 通用、跨平台 | 样式简单 |
| HTML | .html | 样式美观、可打印 | 需要浏览器 |
| Fountain | .fountain | 专业编剧格式 | 需要转换 |
| Final Draft | .fdx | 业界标准 | 需要软件 |

### 10.2 导出为Markdown

**适用场景：** 对话中查看、进一步编辑、复制到其他工具

**操作：**
```
用户：导出为.md

AI：[生成Markdown格式内容]

保存方法：
1. 复制内容
2. 新建文本文件
3. 粘贴内容
4. 保存为.md后缀
```

### 10.3 导出为HTML

**适用场景：** 打印、分享、美观预览

**操作：**
```
用户：导出为HTML

AI：[生成HTML格式内容]

保存方法：
1. 复制全部内容
2. 新建文本文件
3. 粘贴内容
4. 保存为.html后缀
5. 用浏览器打开
```

### 10.4 转换为Word

**方法一：从Markdown转换**
```
Markdown内容 → 复制到Word → 自动转换格式
```

**方法二：从HTML转换**
```
HTML内容 → 浏览器打印 → 另存为PDF
        或
HTML内容 → 复制到Word → 保留基本格式
```

### 10.5 Fountain格式转换

**使用在线工具：**
1. Highland（Mac）
2. Fountainizer（网页工具）
3. WriterSolo（网页工具）

**转换流程：**
```
生成的Markdown → 转换为Fountain格式 → 用专业软件打开
```

```
商业检查
├─ [ ] 类型规则体现？
```

## 12. 实战案例演示

### 12.1 案例：创作一个"失忆特工"故事

#### Step 1: 激活并输入概念

```
用户：救猫咪

AI：你好！我是你的救猫咪编剧助手。
```

## 13. 常见问题与解决方案

### 13.1 技术问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 无法激活 | 触发词不对 | 尝试"救猫咪"或"写剧本" |
| 生成很慢 | 网络问题 | 等待或刷新重试 |
| 内容被截断 | 回复长度限制 | 要求分段生成 |
| 格式混乱 | 输出格式问题 | 要求重新生成指定格式 |

### 13.2 创作问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 类型匹配不准 | 概念描述不清 | 提供更具体的故事描述 |
| 节拍表不合理 | 类型选择不当 | 尝试其他类型或自定义 |
| 角色不立体 | 缺陷设计不够 | 参考角色弧光设计方法 |
| 情节太平 | 冲突不够强 | 加强催化剂、提高压力 |

### 13.3 使用问题

| 问题 | 解决方案 |
|------|---------|
| 想要调整某个节拍 | 直接说"把XX节拍改成XX" |
| 想要不同类型 | 说"我想尝试XX类型" |
| 想要多个选项 | 说"给我3个不同的方向" |
| 想要继续深入 | 说"继续展开" |
