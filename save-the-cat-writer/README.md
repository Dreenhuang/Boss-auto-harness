## 1. 项目概述

### 1.1 项目简介

**救猫咪编剧助手**（Save the Cat! Writer）是一款基于布莱克·斯奈德（Blake Snyder）经典编剧理论体系的AI驱动剧本创作工具。该工具通过智能化的方式，帮助编剧从概念阶段到完整剧本，全流程地应用"救猫咪"编剧方法论。

### 1.2 项目背景

布莱克·斯奈德的《救猫咪》是电影编剧领域最具影响力的方法论之一，其核心理念被好莱坞超过300部院线电影采用。本项目将此理论体系AI化，让更多编剧和爱好者能够便捷地运用这一专业工具。

### 1.3 核心价值

| 价值维度 | 具体说明 |
|---------|---------|
| **结构化创作** | 提供完整的15节拍表框架，确保剧本结构完整 |
| **类型化思维** | 10种电影类型匹配，帮助创作者明确方向 |
| **角色驱动** | 角色弧光设计工具，确保角色转变可信 |
| **案例参考** | 100+经典电影案例，提供学习与灵感来源 |
| **智能辅助** | AI引导的对话模式，降低创作门槛 |

### 1.4 目标用户

| 用户类型 | 使用场景 |
|---------|---------|
| 职业编剧 | 快速生成剧本框架、验证故事结构 |
| 编剧学习者 | 学习救猫咪理论、理解剧本结构 |
| 影视公司 | 项目开发、剧本评估 |
| 内容创作者 | 短视频脚本、故事构思 |
| AI爱好者 | 探索AI+创意工具的结合 |

### 2.2 详细功能

#### 2.2.1 直接生成模式

当用户提供完整的概念时，系统自动执行以下流程：

| 阶段 | 输入 | 输出 |
|------|------|------|
| 概念解析 | 一句话故事描述 | 主角、目标、障碍、类型推荐 |
| 结构生成 | 概念+类型 | 完整15节拍表框架 |
| 场景填充 | 节拍表 | 详细场景描述 |
| 角色设计 | 故事概念 | 主角、B故事角色、反派设定 |
| 剧本生成 | 场景描述 | 完整剧本初稿 |

#### 2.2.2 共创协作模式

通过结构化对话引导用户完善创意：

```
启动访谈 → 分层提问 → 信息整合 → 概念成型 → 结构生成 → 迭代优化 → 最终输出
```

**对话策略特点：**
- 根据用户回答动态调整问题
- 每轮提问后总结已收集信息
- 当信息足够时主动推进下一步
- 用户可以随时要求生成或回顾

#### 2.2.3 智能选项生成

当用户表示没有明确想法时，自动生成3-5个高质量选项：

| 触发词 | 示例 |
|--------|------|
| "我不知道" | 系统自动提供多个故事方向 |
| "没想法" | 展示类型选择菜单 |
| "随便" | 根据用户背景推荐最适合的方向 |
| "你来决定" | 提供3个选项供用户选择 |

## 7. 常见问题解答（FAQ）

### 基础问题

#### Q1: 什么是"救猫咪"？

**答：** "救猫咪"是美国著名编剧布莱克·斯奈德（Blake Snyder）提出的一套电影编剧方法论。其核心理念来自书中一个比喻：剧本开场时，主角需要做一个"救猫咪"的行为——比如救一只猫——来让观众立即喜欢这个角色。

斯奈德将剧本结构化为"15节拍表"，并归纳出"10种电影类型"，帮助编剧系统性地创作出符合观众期待的商业电影。

#### Q2: 我是编剧新手，能用这个工具吗？

**答：** 当然可以！工具设计初衷就是帮助新手理解剧本结构。两种使用模式：

| 模式 | 适合人群 | 使用方式 |
|------|---------|---------|
| 共创协作模式 | 完全新手 | AI引导式提问，逐步完成 |
| 直接生成模式 | 有一定基础 | 直接输入概念，快速生成 |

#### Q3: 生成的内容可以直接用于拍摄吗？

**答：** 生成的内容是高质量的剧本框架和初稿，属于"第一稿"水平。建议流程：

```
生成初稿 → 人工修改 → 打磨细节 → 专业格式 → 投入拍摄
```

### 技术问题

#### Q4: 支持哪些AI助手？

**答：** 理论上支持所有可扩展Skill的AI助手，包括但不限于：
- Claude（Claude Desktop/Claude Code）
- GPT-4（配合插件系统）
- MiniMax
- 其他支持Skill的AI平台

#### Q5: 支持多语言吗？

**答：** 目前主要支持**中文剧本生成**。理论框架和案例库包含中英文电影。用户可以用中文与AI对话，生成中文剧本。

#### Q6: 支持不同长度的剧本吗？

**答：** 支持！默认是110页（标准院线片），但可以指定：

| 长度类型 | 页数 | 适用场景 |
|---------|------|---------|
| 短片 | 5-20页 | 短片、网络视频 |
| 网剧单集 | 30-50页 | 网剧、电视剧 |
| 标准长片 | 90-120页 | 院线电影 |
| 长片+ | 120页以上 | 超长电影 |

### 创作问题

#### Q7: 如何处理类型混合？

**答：** 救猫咪理论允许类型创新。处理方式：

```
主类型（必须遵循）+ 辅助类型（可选添加）

示例：
主类型：推理侦探型（Whydunit）
辅助类型：伙伴之情型（Buddy Love）
结果：《唐人街》式推理+复杂人物关系
```

#### Q8: 角色弧光是什么？

**答：** 角色弧光是角色在故事中的内在转变，包含三要素：

| 要素 | 定义 | 示例 |
|------|------|------|
| 缺陷 | 角色开始时的性格问题 | 过度自信、害怕亲密 |
| 欲望 | 角色追求的外在目标 | 成为英雄、获得爱情 |
| 需求 | 角色真正需要的内在成长 | 学会信任、承担责任 |

#### Q9: 15节拍表必须严格遵守吗？

**答：** 不必。15节拍表是指导方针而非强制规则：

| 情况 | 建议 |
|------|------|
| 新手阶段 | 严格遵循，打好基础 |
| 有经验后 | 可以根据需要调整比例 |
| 创新尝试 | 大胆突破，但要理解规则 |

#### Q10: 如何获取更多案例参考？

**答：** 在使用过程中可以随时询问：

```
"给我更多[类型]的案例"
"分析一下《XX电影》的节拍表"
"这个类型还有哪些经典电影？"
```

### 格式问题

#### Q11: 可以导出为Word格式吗？

**答：** 目前直接支持Markdown、HTML格式。转换方法：

```
Markdown → 复制到Word → 保持格式
HTML → 浏览器打印 → 另存为PDF
```

#### Q12: 什么是Fountain格式？

**答：** Fountain是一种纯文本的剧本格式，被专业编剧软件（如Highland、WriterSolo）支持。优势：
- 纯文本，跨平台兼容
- 格式简单易学
- 可被专业软件识别
